## 摘要
用于具身导航的视觉语言模型（VLM）依赖于从不断增长的轨迹历史中选择固定数量的帧。随着episode的延长，这种选择变得越来越稀疏，然而先前的研究表明，当帧数从8扩展到64时，准确率并未提升，这表明瓶颈不在于帧的数量，而在于表示本身。稀疏的帧选择无法捕捉长时推理所需的结构化行为信号：转向模式、累积位移和路径拓扑。我们提出BIT-Nav（面向导航的脑启发轨迹记忆），这是一个通过紧凑的学习型轨迹记忆来增强冻结VLM导航流程的框架。受海马体路径整合的启发——空间经验被压缩为结构化的episodic痕迹而非存储为原始的感官回放——BIT-Nav在具有相同行为意图的轨迹前缀上，通过多正例InfoNCE对比目标训练一个基于动作和相对位姿序列的Bi-GRU编码器。得到的嵌入通过轻量级MLP投影到VLM token空间，并在每个决策步骤作为单个记忆token注入，从而使模型以恒定的token成本（不受episode长度影响）基于结构化运动历史进行条件推理。

## Abstract
Vision-Language Models (VLMs) for embodied navigation rely on selecting a fixed number of frames from a growing trajectory history. As episodes extend, this selection grows increasingly sparse, yet prior work shows no accuracy gain when scaling from 8 to 64 frames, suggesting the bottleneck is not frame quantity but the representation itself. Sparse frame selection cannot capture the structured behavioral signal that long-horizon reasoning requires: turning patterns, cumulative displacement, and path topology.   We introduce BIT-Nav (Brain-Inspired Trajectory Memory for Navigation), a framework that augments frozen VLM navigation pipelines with a compact learned trajectory memory. Motivated by hippocampal path integration, where spatial experience is compressed into structured episodic traces rather than stored as raw sensory replay, BIT-Nav trains a Bi-GRU encoder over action and relative pose sequences via a multi-positive InfoNCE contrastive objective on trajectory prefixes sharing the same behavioral intent. The resulting embedding is projected into the VLM token space via a lightweight MLP and injected as a single memory token at each decision step, conditioning the model on structured motion history at constant token cost regardless of episode length

### BIT-Nav: Brain-Inspired Trajectory Memory for Embodied Navigation 论文总结

#### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的视觉-语言模型（VLM）在具身导航任务中，处理长轨迹历史的能力存在根本性瓶颈。传统方法通过从不断增长的轨迹历史中选取固定数量的视觉帧来提供历史信息，但随着轨迹长度增加，这种采样变得越来越稀疏。即使将采样帧数从8帧扩展到64帧，导航精度也没有提升。这表明问题不在于帧的数量，而在于**表示形式本身**：稀疏的帧选择无法捕捉长程推理所需的结构化行为信号（如转向模式、累积位移、路径拓扑）。
- **整体含义**：当前VLM在推理层面是“无状态”的，其感知能力与对长期运动历史的结构化理解之间存在鸿沟。作者由此提出应借鉴生物导航中海马体的路径整合机制——即将空间经验压缩为结构化的情景痕迹，而非存储原始感官回放。

#### 2. 方法论

- **核心思想**：BIT-Nav的核心是**将轨迹历史作为VLM的一个显式模态**。它训练一个紧凑的轨迹编码器（BITE），将动作和相对位姿序列编码为固定大小的记忆嵌入，然后通过投影器注入VLM的token空间，使VLM能够基于结构化的运动历史进行条件推理。
- **关键技术细节**：
    1.  **轨迹表示与特征工程**：将原始离散动作（stop, forward, turn-left, turn-right）映射为7维运动特征向量 `mt`，包含累积平面位置`(cx, cy)`、累计航向角`θ`、航向变化`Δθ`，以及前/转向指示器。这使编码器能访问全局几何结构。
    2.  **BITE块 (Bi-GRU Trajectory Encoder)**：使用2层双向GRU处理变长运动特征序列，通过掩码平均池化（masked mean-pool）聚合整个序列，得到512维嵌入`z`，再通过投影头输出128维的L2归一化嵌入。
    3.  **多正例InfoNCE对比学习**：基于“相同起点-终点”的轨迹分组构建minibatch。每个batch包含`B=60`条轨迹（6组×10条/组）。损失函数要求每个锚点与同组中其他9条轨迹（正例）相似，与其余50条轨迹（负例）相异。这迫使编码器学习轨迹的结构不变性（路径拓扑），而非记忆特定路径。
    4.  **与Qwen3-VL-8B集成**：冻结Bi-GRU编码器和VLM。训练一个两层MLP投影器（约4.5M参数），将编码器输出的512维嵌入`z`映射到Qwen的4096维token空间，作为单一token`e_traj`。该token在每个决策步骤与当前视觉、稀疏历史帧及指令token一起送入VLM。
- **公式**：7维特征构建、Bi-GRU双向隐藏状态、掩码平均池化、投影层、多正例InfoNCE损失函数（见论文Eq. 2-15）。

3.  **结构化记忆优于稀疏帧或文本历史**：

#### 7. 优点

- **生物启发且结构简单**：方法受海马体路径整合启发，设计思路清晰。通过对比学习让模型学习路径拓扑的“等价类”，非常巧妙。

- **即插即用、保持VLM能力**：方法不修改VLM架构，通过轻量级投影器实现，对现有VLM导航流程的改动最小，易于集成。
