## 摘要
连续环境下的视觉-语言导航（VLN-CE）要求智能体在未见过的场景中遵循自然语言指令。尽管大型模型（LMs）推动了VLN-CE的发展，但其性能在现实世界的视觉污染下仍严重下降，这是一个关键但尚未充分探索的领域约束。我们引入时间条件流去污器（FlowDec），这是一个专为基于LM的VLN-CE设计的新型图像恢复框架。FlowDec整合了混合时间条件策略，以将生成流路径与历史上下文对齐，并采用动作中心引导过滤来动态评估和整合输出。大量实验表明，FlowDec在导航精度和生成延迟方面均优于最先进的去污方法。我们的方法为在不可预测的现实世界条件下实现鲁棒、高效的具身导航建立了新的范式。

## Abstract
Vision-and-Language Navigation in Continuous Environments (VLN-CE) requires agents to follow natural-language instructions in unseen scenes. While Large Models (LMs) have advanced VLN-CE, their performance remains severely degraded by real-world visual corruptions, a critical yet underexplored domain constraint. We introduce Temporal Conditional Flow Decorruptor (FlowDec), a novel image restoration framework tailored for LM-based VLN-CE. FlowDec integrates a hybrid temporal conditioning strategy to align the generative flow path with historical context and employs action-centroid guided filtering to dynamically assess and integrate outputs. Extensive experiments demonstrate that FlowDec outperforms state-of-the-art decorruption methods in both navigation accuracy and generation latency. Our approach establishes a robust, efficient paradigm for resilient embodied navigation in unpredictable real-world conditions.

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 提出 **FlowDec**（Temporal Conditional Flow Decorruptor），一种基于**条件流匹配（Conditional Flow Matching, CFM）**的轻量级图像去污框架，专门针对LM-based VLN-CE设计。FlowDec在模型-无关（model-agnostic）的前提下，通过历史上下文对齐和动作驱动的质量评估，实现鲁棒、实时、时间一致的去污。

### 关键技术细节
1. **混合时间条件策略（Hybrid Temporal Conditioning）**：
   - 在CFM框架中，定义三种条件类型 \(c_1, c_2, c_3\)：
     - \(c_1 = [z_{c\_aug,1}, \emptyset, 0]\)：仅使用当前帧的增强潜变量（单帧去噪）。
     - \(c_2 = [z_{c\_aug,1}, z_{p\_aug,1} - z_{p\_gt,1}, a_p]\)：引入前一帧的增强-真实潜变量差值与原子动作。
     - \(c_3 = [z_{c\_aug,1}, z_{p\_aug,1} - z_{p\_denoise}, a_p]\)：使用前一帧的去噪潜变量（逐步弥合合成域与真实域差距）。
   - 训练时动态分配条件权重：\(c_1\)权重固定（0.3），\(c_2\)在早期占主导，后期逐渐被\(c_3\)取代，比例由当前epoch控制。

2. **动作中心引导滤波（Action-Centroid Guided Filtering）**：
   - 在训练阶段，从连续两帧真实图像对的潜变量差分中，按原子动作（前进、左转、右转等）计算均值和方差，构建**动作中心** \(A_{gt,a} = \mathcal{N}(\mu_{gt,a}, \sigma_{gt,a})\)。
   - 推理时，模型默认使用 \(c_1\) 生成去污潜变量。对后续帧，计算当前帧与前一帧去污潜变量的差分到对应动作中心的**马氏距离** \(w\)。若 \(w\) 超过阈值 \(w_\theta\)，则触发 \(c_3\) 辅助生成，并根据两个距离加权融合输出。若未超过，则直接输出 \(c_1\) 结果。

3. **推理加速**：将流匹配的起点从纯噪声修改为 \(z_0' = t_\theta z_0 + (1 - t_\theta) z_{c\_cor,1}\)，其中 \(t_\theta = 0.95\)，减少随机性并加速推理（默认使用20步Euler求解器，可降至5–10步）。

### 算法流程
- **训练阶段（Algorithm 1）**：
  1. 编码当前和前一帧的真实图像，构建动作中心。
  2. 对当前和前一帧图像应用数据增强（PIXMIX + SimSiam）模拟污染。
  3. 根据当前epoch采样条件类型，计算CFM损失并更新网络。
  4. 动态更新条件权重（增加\(c_3\)、减少\(c_2\)）。
- **推理阶段（Algorithm 2）**：
  1. 对当前帧输入污染图像，使用起点调整后的噪声初始化潜变量。
  2. 用 \(c_1\) 生成初始去污潜变量。
  3. 若不是第一帧：计算与动作中心的马氏距离 \(w\)；若 \(w > w_\theta\)，用 \(c_3\) 重新生成并计算新距离 \(\tilde{w}\)，融合输出。
  4. 解码为图像，传递给LM-based VLN模型执行动作。

## 6. 主要结论与发现

1. **LM-based VLN模型对视觉污染高度脆弱**：在NaVid上，仅添加散粒噪声即导致SR从41.4%降至约21%，证明鲁棒性问题是真实应用的关键障碍。
2. **FlowDec显著提升鲁棒性**：
   - 在R2R-CE上相对NaVid提升SR 25.33%，在RxR-CE上提升9.38%。
   - 在所有污染类型上超越所有基线（SCUNet、Dec-DPM、Dec-CM），且对清洁输入几乎无性能损失（SR下降<0.5%）。
3. **时间一致性的关键作用**：FlowDec的warp error最低（0.11 vs 基线0.13–0.33），表明帧间一致性比单帧视觉质量对导航更重要。
4. **推理速度快**：比扩散型TTA方法快3–8倍，满足实时导航要求。
5. **真实世界有效**：在四足机器人上，FlowDec在所有四个任务中均优于基线，尤其在光照波动场景下改进更显著。
6. **消融实验揭示设计选择**：混合时间条件训练策略、起点调整（\(t_\theta=0.95\)）、适当滤波阈值（\(w_\theta=0.25\)）均为最佳配置，缺一不可。

## 7. 优点

- **问题新颖且重要**：首次系统研究VLN-CE中的视觉污染鲁棒性问题，填补了领域空白。
- **方法设计巧妙**：
  - 将流匹配应用于VLN任务中的去污，利用条件流匹配实现快速、一步生成。
  - 混合时间条件策略充分利用历史帧信息，同时通过训练动态调整弥合合成-真实域差距。
  - 动作中心引导滤波是一种轻量、可解释的机制，避免误差累积且保持计算高效。
- **模型无关**：FlowDec不依赖具体VLN骨干，可作为即插即用模块，兼容多种LM-based VLN模型（验证了NaVid和Uni-NaVid）。
- **实验全面**：覆盖模拟与真实、多种污染、不同骨干、详细消融，指标丰富（导航、一致性、延迟）。
- **推理效率高**：相比扩散方法，速度优势显著，便于实际部署。

## 8. 不足与局限

1. **实验覆盖仍有局限**：
   - 仅测试了6种主要污染，但真实世界污染分布更复杂、更混合（如同时存在模糊和噪声）。未验证组合污染或未知污染类型。
   - 真实世界实验仅在4个任务、单机器人平台进行，样本量较小（每个任务20次），结论泛化性有限。
2. **对清洁输入有轻微退化**：在干净条件下SR下降约0.5%，虽可接受但存在域差距（合成训练与真实图像域）。论文归因于生成模型的域漂移，但未提出完全消除方案。
3. **动作中心构建依赖大量真实图像对**：需要从训练数据集（70+场景）中收集2.4M图像对来估计稳定的统计量。若应用于全新环境或动作空间不同的机器人，可能需要重新收集或在线估计，增加部署成本。
4. **阈值 \(w_\theta\) 和起点 \(t_\theta\) 为固定超参数**：未提供自适应调整方法，可能在不同环境或污染强度下需要手动调整。
5. **对比方法的公平性讨论**：Dec-DPM和Dec-CM使用相同训练数据与增强，但其原始蒸馏来自Instruct-Pix2Pix（45M图像-文本对）预训练，而FlowDec仅使用R2R-CE/RxR-CE数据，计算资源更少但仍表现更好——这反而是优势，但也说明对比在预训练规模上不对等（论文已指出）。
6. **缺乏对不同LM骨干的广泛验证**：仅验证了NaVid和Uni-NaVid（均来自同一系列），未在更多VLN-CE模型（如NAVILA等）上测试。论文声称模型无关，但证据稍弱。
7. **未分析不同污染强度的影响**：实验采用了固定强度的污染（可能是常见设置），但未做强度敏感性分析。
