## 摘要
构建记忆对于零样本具身导航中的长视距规划至关重要。基于检测器的场景图通常将观测压缩为稀疏节点，丢弃细粒度视觉证据并累积噪声，而基于3D重建的方法在计算上仍显昂贵。我们提出EvoMemNav，一种用于零样本具身导航的高效、自进化、细粒度记忆框架。EvoMemNav构建了视觉-语义记忆图（VSMGraph），将原始视图保留为一等记忆，并通过轻量级语义线索和拓扑关系将其组织成房间-视图-对象层级，保留用于歧义消除和停止验证的细粒度细节。为了扩展至不断增长的记忆，我们引入了有预算的粗到细策略：粗粒度阶段将搜索空间压缩至有希望的区域，细粒度阶段仅针对目标验证和决策调用视觉语言模型。超越静态记忆，EvoMemNav在每个子任务后执行反射驱动的写回，更新附加在图上且编码积累环境知识的先验，以优化未来决策而无需重新训练。在GOAT-Bench和HM3D上的实验，涵盖物体、文本描述和图像目标模态，显示SR/SPL的一致提升，更好的多实例歧义消除，更少的过早停止，以及更强的零样本泛化能力。

## Abstract
Building memory is essential for long-horizon planning in zero-shot embodied navigation. Detector-centric scene graphs often compress observations into sparse nodes, discarding fine-grained visual evidence and accumulating noise, while 3D reconstruction-based methods remain computationally prohibitive. We present EvoMemNav, an efficient, self-evolving, fine-grained memory framework for zero-shot embodied navigation. EvoMemNav constructs a Visual-Semantic Memory Graph (VSMGraph) that keeps raw views as first-class memory and organizes them with lightweight semantic cues and topological relations into a room-view-object hierarchy, preserving fine-grained details for disambiguation and Stop verification. To scale to growing memory, we introduce a budgeted coarse-to-fine policy: a coarse stage compresses the search space into promising regions, and a fine stage invokes a VLM only for targeted verification and decision. Beyond static memories, EvoMemNav performs reflection-driven write-back after each subtask, updating graph-attached priors that encode accumulated environmental knowledge to refine future decisions without retraining. Experiments on GOAT-Bench and HM3D across object, text-description, and image-goal modalities show consistent gains in SR/SPL, with better multi-instance disambiguation, fewer premature stops, and stronger zero-shot generalization.

# 论文总结：EvoMemNav: Efficient Self-Evolving Fine-Grained Memory for Zero-Shot Embodied Navigation

## 1. 核心问题与整体含义

零样本具身导航（Zero-Shot Embodied Navigation）要求智能体在未见过的环境中无需额外训练即可根据自然语言描述、目标图像或类别名称导航至目标。长期规划高度依赖于记忆系统。现有记忆构建方法存在两难：一方面，基于检测器的场景图将观测压缩为稀疏符号节点，丢弃了纹理、空间布局等细粒度视觉证据，且检测噪声会累积到下游推理中；另一方面，基于3D重建的记忆方法计算开销大，难以实时运行且与视觉语言模型（VLM）兼容性差。为此，本文提出 EvoMemNav，一种高效、自进化的细粒度记忆框架，旨在保留原始图像证据的同时结构化组织记忆，并通过有预算的粗到细策略和反思驱动的写回机制实现持续自我改进，无需重训练。

## 2. 方法论

### 核心思想
构建视觉-语义记忆图（VSMGraph），将原始视图作为一等节点保留，并利用轻量级语义标签和拓扑关系组织为房间-视图-对象层次结构。在此基础上，采用预算粗到细策略压缩搜索空间，仅在短列表上调用VLM进行目标选择和停止验证。同时引入反射驱动持续记忆适应（RDCMA），每完成一个子任务后，将交互结果作为图附加先验写回，动态优化后续决策。

### 关键技术细节
- **VSMGraph**：包含房间节点、视图节点（分锚点视图和前沿视图）和对象节点。视图节点存储原始RGB-D图像和位姿。锚点视图来自已探索的、富含对象的区域；前沿视图来自可到达的探索边界。通过导航边和可见性边连接。每个视图附带CLIP房间标签和轻量级对象可见性标签，用于候选分桶。
- **预算粗到细策略**：
  - **粗阶段（Explore）**：利用房间标签和对象标签将锚点视图和前沿视图进行分桶，压缩至预算阈值（KA、KF）。若锚点集为空，直接路由至前沿。
  - **细阶段（Search + Verify）**：对压缩后的候选集调用VLM，输出选择结果及置信度（certain/uncertain/unknown）。若置信度低，回退至探索。到达选定锚点视图后，执行结构化多视图停止验证（Stop verification），返回STOP或RESELECT。若RESELECT，则将视图冷却并重新进入粗阶段。
- **反射驱动持续记忆适应（RDCMA）**：
  - 维护两种时间尺度：Episode-STM（子任务内短期记忆，重置）和Scene-LTM（场景长期记忆，持久保留）。
  - 从导航循环中提取经验（轨迹事件、停止验证结果），写回为图附加先验（如房间支持度、前沿效用/死胡同风险、锚点停止可靠性）。
  - 先验在粗阶段作为探索排序打破平局器，在细阶段作为停止校准提示。
  - 维护机制：指数衰减、top-K上限、冲突抑制。

### 算法流程（文字说明）
1. 每步接收观测，更新VSMGraph（添加新视图、更新对象缓存）。
2. 给定当前目标，粗阶段对锚点视图和前沿视图进行房间感知分桶，选出至多KA个锚点和KF个前沿。
3. 若锚点集非空，细阶段调用VLM从候选集中选择下一目标视图，并输出置信度。
4. 根据置信度决定继续探索还是前往选定锚点。到达锚点后，执行多视图停止验证，决定STOP或RESELECT。
5. 若STOP则任务成功；若RESELECT则将锚点冷却，转回粗阶段。
6. 每个子任务结束后，RDCMA根据轨迹和停止结果更新Episode-STM和Scene-LTM中的先验。

## 3. 实验设计

### 数据集与Benchmark
- **GOAT-Bench**：多模态终身导航基准，包含36个场景，每个场景10个episode，每个episode有5-10个子任务。目标模态包括物体类别名、语言描述和参考图像。评价指标：成功率和SPL。
- **HM3D**：大型3D室内导航基准。HM3Dv1（20场景，2000 episode）和HM3Dv2（36场景，1000 episode），6个物体目标类别。

### 对比方法
训练无关方法：SenseAct-NN Modular、CLIP on Wheels、GOAT、VLMnav、DyNaVLM、TANGO、3D-Mem、MSGNav 等。有训练方法：SenseAct-NN Monolithic、Skill Chain、MTU3D、ZSON、VLFM、SGM 等。

### 主要结果
- GOAT-Bench Val Unseen：EvoMemNav 达到 59.6 SR、38.9 SPL，超越所有训练无关基线（如 MSGNav 52.0 SR 29.6 SPL）。
- HM3Dv1：59.2 SR / 33.6 SPL；HM3Dv2：63.8 SR / 39.4 SPL，均为训练无关方法中最高，部分超越有训练方法。

## 6. 主要结论与发现

- EvoMemNav 在零样本、多模态、终身导航场景下取得显著性能提升，且无需任何训练。
- VSMGraph 通过保留原始视图并组织层次结构，有效缓解了细粒度信息丢失和噪声累积。
- 预算粗到细策略大幅压缩了候选空间，将VLM调用减少41%，同时提高SR 11.2个点。
- 反射驱动写回（RDCMA）可渐进式利用之前子任务的经验，在中间子任务上增益最大，且几乎不增加额外计算开销。
- 结构化多视图停止验证有效减少了同类别错误实例的过早停止问题。

## 7. 优点

- **保留细粒度证据**：将原始图像作为一等节点，避免信息压缩损失。
- **高效的VLM调用**：粗到细预算策略将昂贵推理限制在小规模短列表上，实际VLM调用次数从10.7降为6.5。
- **自进化无需重训练**：RDCMA通过图附加先验持续优化决策，且与粗到细策略正交兼容。
- **多模态泛化能力强**：在同一框架下统一处理物体、文字和图像目标，在两种大型数据集上均达到最佳。
- **结构化消融**：清晰分离了各组件贡献，证明了预算粗阶段是性能提升核心。
- **零样本范围**：完全训练无关，可直接部署至新环境。
