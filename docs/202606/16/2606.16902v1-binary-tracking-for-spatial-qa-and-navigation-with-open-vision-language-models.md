## 摘要
这项工作解决了服务机器人在长距离自我中心路线上的空间问答问题。给定一个查询，例如“在回家的路上哪里可以找到干洗店？”，系统返回一个下游导航组件可以操作的度量坐标。先前的空间问答方法利用基于封闭源模型（如GPT-4o）的检索增强智能体进行路径探索。然而，在现实世界中运行的机器人往往无法可靠地依赖在线封闭源模型，因为网络不稳定、通信延迟和部署成本。这产生了对基于开源的空间问答方法的需求，这些方法可以在机器人上运行，但先前在这方面的研究仍然有限。这项工作提出了BinTrack，一个简单而有效的、完全开源的空间定位智能体，它利用了机器人轨迹的时间顺序。BinTrack在从查询中识别出的两个锚点地标之间的轨迹段上执行二分搜索。它相比其他开源实现将整体准确率提高了高达22.8%，甚至在SpaceLocQA基准的全局类别上匹配了报告中的封闭源模型结果，这是迄今为止需要强大推理智能体（如GPT-4o）的最具挑战性的设置。此外，其优化的推理策略始终比先前方法带来超过1.5倍的推理加速。最后，这项工作发布了GangnamLoop，这是一个新颖且实用的多行程户外基准，通过在公共街道上部署真实四足机器人并采用匿名化策略收集而成。它重新访问了不同户外条件下的相同位置，并将机器人的低视角与人类主人的视角配对。源代码和数据集公开发布于https://github.com/ndb796/BinaryTracking。

## Abstract
This work addresses spatial question answering for service robots traversing long egocentric routes. Given a query such as "where can I find a dry cleaner on the way back home?", the system returns a metric coordinate that downstream navigation components can act on. Prior Spatial Question Answering approaches leverage retrieval-augmented agents built on closed-source models such as GPT-4o for path exploration. However, robots operating in the real world often cannot reliably depend on online closed-source models due to network instability, communication latency, and deployment cost. It creates a need for open-source based Spatial Question Answering approaches that can run onboard the robot, yet prior research in this direction remains limited. This work proposes BinTrack, a simple yet effective, fully open-source spatial-localization agent that leverages the temporal ordering of a robot's trajectory. BinTrack performs a binary search over the trajectory segments between two anchor landmarks identified from a query. It improves overall accuracy by up to 22.8% over other open-source implementations and even matches the reported closed-source model result on the global category of the SpaceLocQA benchmark, the most challenging setting that has so far required strong reasoning agents such as GPT-4o. Furthermore, its optimized inference strategy consistently yields more than a 1.5x inference speedup over previous approaches. Finally, this work releases GangnamLoop, a novel and practical multi-trip outdoor benchmark collected by deploying a real quadruped robot on public streets with the anonymization policy. It revisits the same locations under different outdoor conditions and pairs the robot's low viewpoint with the human owner's. The source codes and datasets are publicly available at https://github.com/ndb796/BinaryTracking

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：服务机器人在长程自我中心（egocentric）轨迹上回答空间问题（Spatial Question Answering, SQA）时，需要从长期视频和里程计中检索目标坐标。现有最先进方法（如Meta-Memory、ReMEmbR）依赖闭源大模型（如GPT-4o）作为规划代理和视觉验证器，导致：
  - 网络不稳定、通信延迟导致实时性差；
  - 频繁调用闭源API产生高昂运营成本；
  - 封闭生态阻碍可重复性和自主部署。
- **动机**：迫切需要**完全开源的SQA系统**，能够在机器人本地运行，同时保持高精度。然而，直接替换闭源模型为同等规模的开源模型（如Qwen2.5-32B）会导致性能急剧下降，尤其是在需要多步推理的**全局查询**（global query）上（准确率下降29.6%）。
- **核心洞见**：机器人轨迹天然形成按时间排序的序列，查询中两个锚点地标定义了连续子区间，将目标定位简化为**一维搜索问题**。

## 2. 论文提出的方法论
### 核心思想：BinTrack – 二进制跟踪（Binary Tracking）
- **基本流程**：
  1. 将轨迹分割为固定时长（1.5秒）的片段，每个片段用开放VLM（Qwen2.5-VL-7B）生成**三种视角**的标题（全景、中心、细节），并存储为向量数据库。
  2. 给定查询，提取目标实体Z和两个锚点地标X、Y（如“从办公室到地铁站”），找到锚点对应的轨迹片段索引iX、iY。
  3. 在区间[iX+1, iY-1]上进行**二分搜索**：每次比较左半区间和右半区间的语义证据分数（evid = α·均值 + β·最大值），保留分数更高的半区间，直到区间长度 ≤ k_leaf（k_leaf=16）。
  4. 在最终叶区间内，调用视觉验证器（verifier）从top-k候选中选出最终片段，输出其坐标。
- **关键技术细节**：
  - **多视角记忆**：每个片段生成三类标题（full/center/detail），弥补7B级VLM细节捕捉能力不足。
  - **证据分数**：`evid(区间) = α·mean(σ(i)) + β·max(σ(i))`，其中σ(i)为标题嵌入与查询的余弦相似度。
  - **复杂度**：二分搜索调用次数为O(log n)，仅调用一次视觉验证器，大幅降低耗时（验证器成本远高于检索）。
  - **强化机制**：
    - **锚点感知验证池**：验证池混合语义相似候选和几何邻近候选，避免仅依赖语义。
    - **锁定-Y重排序**：对含有“最近到Y”的查询，先定位Y再按距离重排序Z候选。
- **智能体循环**：使用开源语言模型Qwen2.5-32B-AWQ作为规划代理，在少量工具（语义检索、空间范围检索、二叉跟踪、最终输出）中选择动作，支持4步预算。

## 6. 论文的主要结论与发现
1. **BinTrack在开源约束下显著优于所有开源基线**，在SpaceLocQA上整体准确率67.4%（比开源Meta-Memory高22.8个百分点），甚至在**全局类别上与GPT-4o闭源版本持平（62.2%）**。

## 7. 优点
- **完全开源**：所有组件均为开源模型，无需任何API依赖，可完全在机器人本地运行。
- **结构简单高效**：利用轨迹天然的时间排序，将复杂图搜索简化为二分搜索，降低计算负担。
- **贡献显著**：在全局查询上匹敌GPT-4o结果，这是之前认为只有强封闭模型才能解决的类别。
- **数据贡献**：发布GangnamLoop——首个多行程室外四足机器人SQA基准，支持光照变化、重复访问、双视角评估，促进实用研究。
- **消融与失败分析详尽**：提供对设计选择和失效原因的深度洞察，为后续改进指明方向。
- **低资源适应性**：提供7B级低VRAM配置（20GB），在资源受限的机器人上也适用。

## 8. 不足与局限
1. **离线记忆构建**：标题生成和索引仍需在遍历新路线后离线完成（GPU时长数小时），无法实现完全在线实时部署。
2. **跨轨迹整合缺失**：简单的跨轨迹拼接检索没有带来收益，论文承认需要更精致的对齐方法（未来工作）。
3. **验证器/检索局限**：7B级验证器对小型物体、不常见文字（韩语）识别率低，导致大量假阴性失败。32B级验证器受GPU显存限制未使用。
4. **代理推理不完善**：在GangnamLoop上只有不到一半的全局查询实际调用了二叉跟踪，代理经常跳过关键工具直接输出答案，说明提示工程仍需改进。
5. **地面真值依赖单一阈值**：仅使用τ=15m一个主要阈值，虽提供了其他阈值的曲线，但实际应用中对精度要求可能更严格。
6. **场景与任务范围**：专注于城市街道和特定路线，未覆盖更加杂乱的室内环境或开放式导航任务。
7. **计算资源需求**：尽管提供低VRAM版本，但达到最佳性能仍需双RTX 6000 Ada（48GB）GPU，可能限制小规模团队复现。
