## 摘要
构建记忆对于零样本具身导航中的长期规划至关重要。以检测器为中心的场景图通常将观测压缩为稀疏节点，丢弃细粒度视觉证据并积累噪声，而基于3D重建的方法计算成本高昂。我们提出EvoMemNav，一种高效、自演化、细粒度的记忆框架，用于零样本具身导航。EvoMemNav构建了一个视觉语义记忆图（VSMGraph），将原始视图作为第一类记忆，并通过轻量级语义线索和拓扑关系将其组织成房间-视图-对象层次结构，保留细粒度细节以用于消歧和停止验证。为了扩展到不断增长的记忆，我们引入了一种预算约束的粗到细策略：粗阶段将搜索空间压缩为有前景的区域，细阶段仅调用VLM进行目标验证和决策。除了静态记忆外，EvoMemNav在每个子任务后执行反思驱动的回写，更新附加到图的先验知识，编码累积的环境知识以优化未来决策而无需重新训练。在GOAT-Bench和HM3D上的物体、文本描述和图像目标模态实验显示，在SR/SPL上持续提升，具有更好的多实例消歧、更少的过早停止和更强的零样本泛化能力。

## Abstract
Building memory is essential for long-horizon planning in zero-shot embodied navigation. Detector-centric scene graphs often compress observations into sparse nodes, discarding fine-grained visual evidence and accumulating noise, while 3D reconstruction-based methods remain computationally prohibitive. We present EvoMemNav, an efficient, self-evolving, fine-grained memory framework for zero-shot embodied navigation. EvoMemNav constructs a Visual-Semantic Memory Graph (VSMGraph) that keeps raw views as first-class memory and organizes them with lightweight semantic cues and topological relations into a room-view-object hierarchy, preserving fine-grained details for disambiguation and Stop verification. To scale to growing memory, we introduce a budgeted coarse-to-fine policy: a coarse stage compresses the search space into promising regions, and a fine stage invokes a VLM only for targeted verification and decision. Beyond static memories, EvoMemNav performs reflection-driven write-back after each subtask, updating graph-attached priors that encode accumulated environmental knowledge to refine future decisions without retraining. Experiments on GOAT-Bench and HM3D across object, text-description, and image-goal modalities show consistent gains in SR/SPL, with better multi-instance disambiguation, fewer premature stops, and stronger zero-shot generalization.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：零样本具身导航（zero-shot embodied navigation）中，智能体需要在未见过的环境中根据多模态指令（物体标签、文本描述、参考图像）导航到指定目标。长期规划依赖记忆系统，但现有方法存在权衡困境：
  - **检测器场景图**：将观测压缩为稀疏符号节点，丢弃了纹理、空间布局等细粒度视觉信息，且对检测噪声敏感，累积错误；
  - **3D重建记忆**：计算开销高，实时性强且与VLM（视觉-语言模型）兼容性差；
  - **图像缓存**：视图缺乏结构化组织，易导致过早停止、同类别目标混淆，且记忆不会自我改进。
- **研究含义**：提出一种兼顾效率与细节、具备自进化能力的记忆框架，在不重新训练的情况下提升导航鲁棒性与泛化能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 整体框架（EvoMemNav）
- 输入：多模态目标 g（物体标签/文本/图像）和连续RGB-D观测流；
- 核心组件：**视觉语义记忆图（VSMGraph）** + **预算约束粗到细策略** + **反思驱动连续记忆适应（RDCMA）**。

### 2.2 VSMGraph 构建
- **图结构**：房间-视图-对象三层层次图，节点包括：
  - **视图节点（View Nodes）**：分为**锚点视图（Anchor Views）**（存储原始RGB图像+位姿）和**前沿视图（Frontier Views）**（表示未探索边界）；
  - **对象节点（Object Nodes）**：轻量级实例假设（类别、3D位置），仅用于候选分组标签，不压缩记忆；
  - **房间节点（Room Nodes）**：通过CLIP对视图图像分类获得离散房间标签。
- **边**：可导航边（连续无碰撞视角间）和可见性边（视图到其观察到的对象）。
- **特点**：保持原始视图作为“一等公民”，所有停止决策基于多视图图像验证，而非依赖检测器输出。

### 2.3 预算约束粗到细导航策略
- **粗阶段（Explore）**：
  - 利用房间标签和对象命中标签对候选锚点视图和前沿视图进行**分桶（bucketing）** + Top-K压缩，得到紧凑候选集（|CA|≤KA，|CF|≤KF）；
  - 若锚点集为空，直接路由到前沿视图探索。
- **细阶段（Search + Verify）**：
  - **Search**：仅对压缩候选集调用VLM，选择下一目标视图；若决策置信度低（uncertain/unknown），则回退到探索；
  - **Verify**：到达选定锚点视图后，进行**结构化多视图停止验证**，返回STOP或RESELECT；RESELECT触发冷却并重新进入粗阶段。
- **恢复机制**：当重复选择或死锁时，强制进入前沿探索。

### 2.4 反思驱动连续记忆适应（RDCMA）
- **动机**：使记忆能积累经验，避免重复错误，无需重训练。
- **机制**：每个子任务结束后，将交互结果（访问房间、选择的锚点/前沿、停止验证结果）写成图附加先验：
  - **Episode-STM**（短期记忆）：当前回合内抑制和循环避免；
  - **Scene-LTM**（长期记忆）：跨子任务持久化，带衰减和Top-K裁剪。
- **使用**：在Explore中作为排序调解器（优先选择高历史效用、低死胡同风险的候选）；在Verify中提供停止可靠性提示，校准决策。

## 7. 优点

1. **创新方法**：首次将**原始视图作为第一类记忆**组织成层次图，在保留细粒度信息的同时实现高效检索和验证；
2. **避免对检测器的过度依赖**：对象线索仅作为软标签，核心决策基于多视图图像，鲁棒性更强；
3. **计算高效**：粗到细策略显著减少VLM调用次数，是**训练无关**方法中最优的性价比；
4. **自进化能力**：RDCMA无需重新训练即可利用先前经验优化探索和停止决策，弥补静态记忆的不足；
5. **泛化性强**：在多个基准（GOAT、HM3D）和多模态（物体/语言/图像）上均有一致提升。
