## 摘要
当初始用户请求未能唯一指定目标实例时，自然语言实例导航变得具有挑战性。一个实用的智能体应通过主动询问仅需区分目标与相似干扰物的信息来减轻用户负担，而非要求事先提供详细描述。现有方法往往未能实现这一目标：它们可能在充分探索替代方案之前就止步于第一个貌似合理的候选对象，或者即使收集了多个候选对象，也只是询问从单个候选对象导出的目标属性，而非选择旨在区分候选池中对象的问题。结果，尽管进行了对话，智能体仍可能无法将目标与干扰物区分开，导致过早决策和冗长的用户回应。我们提出主动式实例导航与比较判断（ProCompNav），这是一个两阶段框架，首先构建候选池，然后通过比较判断识别目标。在每一轮中，ProCompNav提取一个属性-值对，该对能分裂当前候选池，提出一个二分的是/否问题，并立即剪枝所有不一致的候选对象。这将歧义消解从开放式的目标描述重新定义为池级判别性提问，其中每个问题都经过选择以缩小候选集。在CoIN-Bench上，ProCompNav相比具有相同最小输入的交互式基线和具有详细描述的非交互式基线，提高了成功率，同时大幅缩短了响应长度。ProCompNav还在TextNav上取得了最先进的成功率，表明比较判断对于相似干扰物中的实例级导航具有广泛用途。

## Abstract
Natural-language instance navigation becomes challenging when the initial user request does not uniquely specify the target instance. A practical agent should reduce the user's burden by actively asking only the information needed to distinguish the target from similar distractors, rather than requiring a detailed description upfront. Existing approaches often fall short of this goal: they may stop at the first plausible candidate before sufficiently exploring alternatives, or, even after collecting multiple candidates, ask about the target's attributes derived from individual candidates rather than questions selected to distinguish candidates in the pool. As a result, despite the dialogue, the agent may still fail to distinguish the target from distractors, leading to premature decisions and lengthy user responses. We propose Proactive Instance Navigation with Comparative Judgment (ProCompNav), a two-stage framework that first constructs a candidate pool and then identifies the target through comparative judgment. At each round, ProCompNav extracts an attribute-value pair that splits the current pool, asks a binary yes/no question, and prunes all inconsistent candidates at once. This reframes disambiguation from open-ended target description to pool-level discriminative questioning, where each question is chosen to narrow the candidate set. On CoIN-Bench, ProCompNav improves Success Rate over interactive baselines with the same minimal input and non-interactive baselines with detailed descriptions, while substantially reducing Response Length. ProCompNav also achieves state-of-the-art Success Rate on TextNav, suggesting that comparative judgment is broadly useful for instance-level navigation among similar distractors.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在自然语言驱动的实例导航任务中，用户初始查询往往模糊不清（例如仅说“找柜子”），智能体需要通过人机对话从同类别多个相似实例中区分出用户真正意图的目标。现有方法存在两大缺陷：
  - 过早决策：智能体可能遇到第一个看似匹配的干扰物就停止，未能充分探索其他候选。
  - 提问缺乏区分性：即使收集了多个候选，提出的问题（如“目标是什么颜色？”）不是基于候选之间的对比来设计的，导致无法有效消除歧义，同时用户需要提供冗长的描述性回答。

- **研究动机**：设计一种能够主动、高效地将模糊查询转化为明确目标的导航框架，最小化用户负担（少提问、短回答）并提高成功率。

- **整体含义**：将歧义消解从“开放式目标描述”重新定义为“池级判别性提问”，通过构建候选池并进行比较判断，每轮只需回答一个二分问题即可大幅缩小候选范围。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
**ProCompNav**（Proactive Instance Navigation with Comparative Judgment）采用两阶段流水线：
1. **候选池构建阶段**：在未知环境中探索，收集所有同类别实例，形成多视图候选池。
2. **递归比较阶段**：通过比较候选之间的属性差异，提出二分问题，逐步剪枝候选池直至唯一目标。

### 关键技术细节
- **候选池构建**：
  - 利用Visual Language Frontier Maps (VLFM) 探索。
  - 检测到类别实例后，提取3D点云并合并到已有候选（基于重叠比例），否则新建候选。
  - 对每个候选，从多个视角中聚类选择6个代表性视图，拼成多视图拼图，用多模态大语言模型（MLLM）生成综合描述（包含属性-值对）。

- **递归比较判断（RCJ）**：
  - 初始化候选池 \(U_0 = C\)。每轮 \(t\)：
    1. **相似性核心集选择**：从 \(U_t\) 中选取一个语义相似的核心集 \(G_c\)，其余为补集 \(G_r\)。使用贪心剥离算法最大化核心集内部平均相似度（融合文本和视觉嵌入）。
    2. **判别性属性（DA）发现**：从 \(G_c\) 的描述中提取候选属性集合，用自然语言推理（NLI）模型计算每个属性对核心集和补集的蕴含分数，选择分数差距最大的属性 \(a_t^*\)（在核心集中普遍存在，在补集中普遍不存在）。
    3. **属性引导的组精化**：将补集中实际也包含该属性的候选移回核心集（基于NLI得分阈值 \(\tau=0.9\)）。
    4. **交互式剪枝**：向用户提问“目标是否具有 \(a_t^*\)？”，用户回答是/否。根据回答将下一轮候选池设为 \(G_c\) 或 \(G_r\)。
    5. 重复直至 \( |U_t| = 1 \)。

- **细节**：
  - 若用户回答“否”但补集为空，则说明候选池可能不包含目标，恢复探索增加候选。
  - 使用DeBERTa-v3-large作为NLI验证器。
  - 循环检测和360°旋转启发式改进探索效率。

## 7. 优点

- **创新性**：将实例导航中的人机交互从“描述目标”范式转变为“比较候选”范式，提出递归比较判断，概念新颖且实用。
- **效率与用户友好**：只需回答二分问题（是/否），极大降低用户负担；每轮可一次性剪除多个候选，减少交互轮次。
- **鲁棒性**：即使在候选高度相似时也能通过对比属性逐步缩小范围，不要求属性唯一标识目标。
- **普适性**：方法设计适用于交互（CoIN）和非交互（TextNav）场景，迁移简单。
- **零样本免训练**：无需训练导航策略，利用预训练模型即可工作，易于部署。
- **实验设计清晰**：消融、分析、计算成本、定性对比完备，结论可靠。

## 8. 不足与局限

- **模拟环境局限**：所有实验在仿真环境（CoIN-Bench、TextNav）中进行，未在真实机器人平台上验证，可能存在sim-to-real gap。
- **用户模拟器偏差**：使用MLLM作为用户模拟器，其行为可能与真实人类用户有差异，尤其对二分问题的容忍度和理解能力。真人对用户负担的感知可能不同。
- **候选池构建效率**：需要至少收集 \(N_{min}=5\) 个候选才能启动比较阶段。在同类实例稀少的环境中，可能需要长时间探索或无法构建池，导致失败。
- **动态调整不足**：候选池大小阈值固定，缺乏自适应策略。未来工作可探索动态池大小。
- **计算资源**：虽然推理时间减少，但仍需要两个RTX 3090 GPU，对资源有限的平台可能依然过重。
- **探索启发式**：添加的循环检测和360°旋转是轻量级启发式，虽有效但非最优可能影响在更复杂环境下的探索效率。
