## 摘要
具身导航要求智能体将语言和视觉观测映射为一系列空间动作，以驱动真实机器人在从未见过的环境中移动。主流方法是在不断扩大的机器人轨迹数据集上扩展视觉-语言-动作（VLA）基础模型。本文认为，对于导航而言，通用性可以通过结构获得，而不仅仅依赖于数据规模。导航的底层决策结构可简化为单一的语言-视觉-机器人动作翻译。语言动作发出语义级的方向指令，视觉动作发出像素级的视觉目标。这两个输出都位于预训练多模态大语言模型（MLLM）的自然输出流形内，因此任务可以由智能体推理完成，而无需从机器人数据中学习。因此，我们提出了Uni-LaViRA，这是一种统一的智能体架构，将相同的见解以零样本方式扩展到四个任务族（VLN-CE、ObjectNav、EQA和Aerial-VLN）以及四种异构真实机器人（轮式、四足、人形机器人和自建无人机）。两种智能体循环机制使这种统一成为现实。待办事项列表记忆（TDM）每一步重写一个结构化的待办子目标清单，将未完成的项目重新纳入智能体最近的注意力窗口。二次机会回溯（SCB）将机器人回滚到错误发生前的状态，并基于失败的子轨迹调整智能体的下一步计划，将单次导航转变为自我纠正的过程。无需任何训练，Uni-LaViRA在VLN-CE R2R上达到60.7%的SR，在VLN-CE RxR上达到51.3%，在HM3D-v2上达到77.7%，在HM3D-OVON上达到60.0%，在MP3D-EQA上达到54.7%，在OpenUAV上达到40.0%，匹配甚至超越了近期消耗数百万样本和数千GPU小时的训练导航基础模型。

## Abstract
Embodied navigation requires an agent to map language and visual observations to a stream of spatial actions that drive a real robot through environments it has never seen. The dominant approach has been to scale vision-language-action (VLA) foundation models on ever-larger collections of robot trajectories. This paper argues that, for navigation specifically, generality can be obtained structurally, not only through data scale. The underlying decision structure of navigation reduces to a single Language-Vision-Robot Actions Translation. The language action emits semantic-level directional command and the vision action emits a pixel-level visual target. Both outputs lie inside the natural output manifold of pretrained multimodal large language models (MLLMs), so the task can be reasoned about by an agent rather than learned from robot data. Therefore, we present Uni-LaViRA, a unified agentic architecture that extends the same insight to four task families (VLN-CE, ObjectNav, EQA, and Aerial-VLN) and to four heterogeneous real robots (Wheeled, Quadruped, Humanoid robot, and a self-built UAV) in a zero-shot manner. Two agent-loop mechanisms make this unification practical. TODO List Memory (TDM) rewrites a structured checklist of pending sub-goals at every step, reciting the unfinished items back into the agent's most recent attention window. Second Chance Backtrack (SCB) rolls the robot back to the pre-error state and conditions the agent's next plan on the failed sub-trajectory, turning single-pass navigation into a self-correcting process. With zero training effort, Uni-LaViRA reaches 60.7% SR on VLN-CE R2R, 51.3% on VLN-CE RxR, 77.7% on HM3D-v2, 60.0% on HM3D-OVON, 54.7% on MP3D-EQA, and 40.0% on OpenUAV, matching or even surpassing recent training navigation foundation models that consume millions of samples and thousands of GPU-hours.

## 2. 方法论

### 核心思想
将导航决策分解为三个层次：
- **语言动作（Language Action）**：由MLLM（Gemini-3.1-Pro）输出语义级方向指令，如 `turn(front/left/right/back)`，以及停止、回溯等动作。
- **视觉动作（Vision Action）**：由MLLM（Qwen3.5-27B）在选定视图上输出像素级目标框/点，通过深度图反投影得到3D位置。
- **机器人动作（Robot Action）**：确定性几何控制器（Fast-Marching规划器）将3D目标点转换为具体控制指令。

### 关键技术细节
- **TODO List Memory (TDM)**：每步维护一个结构化子目标清单（内容、状态、结果），智能体在决策前更新清单，保证长指令下的计划连贯性。
- **Second Chance Backtrack (SCB)**：当发现错误时，将机器人回滚到先前正确位置，并将失败子轨迹作为证据输入MLLM重新规划，实现自校正。
- **公式表示**：  
  `At = πrobot( φvis( φlang(T, Ot, Ht) ), Dt, K, poset )`  
  其中 `φlang` 输出方向+辅助字段，`φvis` 输出边界框，`πrobot` 为确定性控制器。

### 算法流程（文字说明）
1. 初始化TDM清单；  
2. 每步调用 `φlang` 同时输出TDM更新和动作选择；  
3. 若动作为 `backtrack`，则执行SCB（回滚+基于失败轨迹重新规划）；  
4. 若动作为 `stop`，终止；  
5. 否则调用 `φvis` 在选定视图上定位目标，控制器执行短时轨迹；  
6. 更新观测和历史。

## 4. 资源与算力

- **训练模型对比**：NavFoM需要56个H100 GPU运行72小时（约4,032 GPU-hours），OmniNav消耗14,592 GPU-hours，ABot-N0消耗6,144 GPU-hours。
- **Uni-LaViRA**：**完全零训练**，所有MLLM以API方式调用（Gemini-3.1-Pro, Qwen3.5-27B）。仿真实验在8张NVIDIA RTX 4090上并行运行。
- **推理成本**：每次语言动作调用平均22k输入+1.6k输出token，约9.6 PFLOPs；视觉动作调用约1k输入+0.1k输出token，约59 TFLOPs。完整六任务评估（600条轨迹）约需5-7小时（40个并行API worker）。

## 5. 实验数量与充分性

- **主实验**：六个基准数据集上，每个任务三个种子，共18组指标。
- **消融实验**：对比Full、w/o TDM、w/o SCB、w/o both四种配置，六个任务共24组。

## 6. 主要结论与发现

- **零训练达到SOTA**：一个统一的零样本智能体架构在六个基准上超越或匹配大量训练的基础模型（如HM3D-v2 SR 77.7% vs 训练模型FiLM-Nav 77.0%）。
- **结构通用性成立**：导航任务的动作空间确实位于MLLM输出流形内，无需训练即可利用其推理能力。
- **TDM与SCB协同有效**：两者结合比单独使用提升显著（平均SR提升6-11个点），TDM主导长指令任务，SCB主导室内目标搜索任务。
- **跨本体迁移简单**：只需更换底层控制器和校准传感器（约10小时），上层MLLM字节相同。

## 7. 优点

- **零训练无需机器人数据**：极大降低具身导航研究的门槛，无需大规模GPU集群。
- **统一框架覆盖广**：单方案覆盖室内导航、室外飞行、问答等多种任务，且支持轮式、四足、人形、无人机异构本体。
- **结构设计简洁优雅**：将复杂导航分解为自然语言、像素级视觉和几何控制，三者互为补充。
- **自我修正能力**：TDM和SCB使智能体具有明确计划和不依赖记忆遗忘的纠错机制，可解释性强。
- **推理成本可控**：仅需API调用，实际部署成本远低于端到端训练。

## 8. 不足与局限

- **依赖闭源MLLM**：最强骨干Gemini-3.1-Pro为闭源，开源模型差距明显，限制了自托管能力。
- **长指令任务仍有差距**：在RxR（120词平均指令）上SR 51.3%，落后领先训练模型约10个点，说明纯推理方法在高复杂度指令下仍有局限性。
- **动态障碍处理不足**：仅依靠底层控制器反应式避障，未在语言动作层整合人类/动态物体意图推理。
- **EQA视觉识别错误**：最终答案正确率受限于MLLM的细粒度视觉判别能力（如黑色与深棕混淆）。
- **实验覆盖**：真实部署仅有定性验证，缺乏定量指标；部分仿真实验使用100条子集而非全量，虽经验证偏差可控，但极少数高方差指标（如OpenUAV）存在偏差。
- **计算成本向API转移**：虽然训练成本为零，但推理成本依赖于商业API，长时间运行可能产生较高费用，且网络延迟影响实时性（尤其是无人机场景）。
