## 摘要
本文介绍了为CMU视觉-语言-动作（VLA）挑战赛设计的集成系统，旨在使自主智能体能够基于自然语言指令执行复杂任务。我们的框架采用模块化架构，协调环境映射、问题处理和导航。系统运行于两个并行流：一个感知管道，利用OwlViT嵌入从实时摄像头流构建语义体素地图；另一个语言管道，使用视觉-语言模型对用户命令进行分类。地图构建受时间限制；若达到500秒探索限制，系统将使用部分地图继续执行。随后，分类后的查询基于地图的几何和语义上下文进行接地，为VLM生成详细提示。这产生可执行的输出，展示了弥合人类语言与机器人动作之间差距的有效解决方案。

## Abstract
This paper presents an integrated system for the CMU Vision-Language-Action (VLA) Challenge, designed to enable an autonomous agent to perform complex tasks based on natural language instructions. Our framework employs a modular architecture that orchestrates environment mapping, question processing, and navigation. The system operates in two parallel streams: a perception pipeline that constructs a semantic voxel map from real-time camera feeds using OwlViT embeddings, and a language pipeline that classifies user commands with a Vision-Language Model. The mapping is time-constrained; the system proceeds with a partial map if a 500-second exploration limit is reached. The classified query is then grounded in the geometric and semantic context of the map to generate a detailed prompt for the VLM. This yields an actionable output, demonstrating a capable solution for bridging the gap between human language and robotic action.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
采用**模块化架构**，并行运行两条独立管道：
- **感知管道**：利用OwlViT从实时摄像头流构建**语义体素地图**，存储几何信息与语义嵌入。
- **语言管道**：使用**视觉-语言模型（VLM，Gemini 2.0 Flash）** 对用户指令进行分类、提取关键对象，并与地图上下文融合生成详细提示，最终由VLM输出可执行动作。

### 关键技术细节

1. **自主探索与2D网格地图**：
   - 基于LiDAR数据构建2D占用网格地图。
   - 采用**全覆盖探索策略**：采样可通行点（间距≥1.25m）、计算最优路径（访问所有点）、由路径点控制器顺序执行。
   - 探索阶段受**500秒时间限制**；超时则使用部分地图继续。

2. **语义体素地图构建**：
   - 使用**google/owlvit-base-patch32**对每一帧检测到的物体生成特征嵌入。
   - 将嵌入、3D坐标、尺度信息逐步集成到**体素地图**中（体素大小0.15m）。
   - 地图是**增量式**的，每次感知循环更新。

3. **VLM查询处理**：
   - **查询分类**：Gemini 2.0 Flash将指令分为“数值型”、“对象引用型”、“指令跟随型”三类。
   - **对象接地**：利用查询名词的文本嵌入与体素地图中所有嵌入计算相似度，定位3D点。
   - **上下文提示生成**：将对象名称、坐标、尺度、2D地图位置格式化为详细提示，发送给VLM进行推理。

4. **动作生成**：
   - 数值型：直接返回整数。
   - 对象引用型：在RViz中高亮标记。
   - 指令跟随型：解析VLM输出的高层计划（如“Goto(x,y), Avoidbetween(...)”），动态修改2D网格，计算路径并执行。

### 优化措施
为满足10分钟总时间限制，作者进行了速度优化：
- 增加采样点间距（0.6m→1.25m）
- 放宽路径点到达条件（0.65m→1.0m）
- 增大体素尺寸（0.15m），降低计算负载，将探索时间从8分42秒降至4分17秒。
